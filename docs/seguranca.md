# Seguranca

## English

SentryWard is defensive local security tooling for authorized project analysis. The MVP uses deterministic rules and local heuristics only. It must not upload source code, require telemetry, run exploit automation, or scan third-party systems.

## Portugues

SentryWard e uma ferramenta defensiva de seguranca local para analise autorizada de projetos. O MVP usa apenas regras deterministicas e heuristicas locais. Ele nao deve enviar codigo-fonte, exigir telemetria, executar automacao ofensiva ou escanear sistemas de terceiros.

## Espanol

SentryWard es una herramienta defensiva de seguridad local para analisis autorizado de proyectos. El MVP usa solo reglas deterministicas y heuristicas locales. No debe subir codigo fuente, exigir telemetria, ejecutar automatizacion ofensiva ni escanear sistemas de terceros.

## Scope

- Local files, local config, local dependencies, local git state, and explicitly provided local development URL.
- Deterministic static guards for secrets, browser exposure, routes, auth, database, uploads, CORS/CSRF, sessions, webhooks, Docker, CI, dependencies, prompt injection, Supabase, and Firebase.
- Optional future AI provider interface may be documented as planned but must remain disabled and unimplemented.
- Optional Sema governance events generated by SentryWard findings. These events are local files in the MVP and can be consumed by agents or the Sema CLI.

## Procedure

- Keep findings evidence-based and deduplicated with stable fingerprints.
- Use confidence, severity, impact, recommendation, and references to avoid noisy alerts.
- In watch mode, interrupt only for High or Critical findings.
- Safe fixes require confirmation; secret rotation is always a manual step when exposure happened.
- SentryWard must work without Sema. When the person or team wants semantic governance, governed mode checks for an applicable `.sema` contract, drift, and impact. If strict contract mode is requested and no applicable contract exists, the fix is blocked; otherwise SentryWard reports that Sema is a recommended complement and continues with the non-governed safe flow.

## Validation

- `pnpm build`, `pnpm lint`, and `pnpm test` must pass.
- Smoke checks must verify help, init, scan, localized scan, explain, HTML report, and JSON report.
- Reports must not include full secret values.

## Rollback

- Disable or revert any guard that creates high-severity false positives without clear evidence, exposes secrets in output, or performs network behavior outside explicit local URL checks.
